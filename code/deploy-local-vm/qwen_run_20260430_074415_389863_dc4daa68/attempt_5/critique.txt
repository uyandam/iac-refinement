The playbook fails because the `focal_servers` group is not defined in your Ansible inventory. 

**Failing Task:**  

* All tasks within the play "Install kubeadm on Ubuntu Focal" are skipped because no hosts match the pattern `focal_servers`.


**Error:** 

* "[WARNING]: Could not match supplied host pattern, ignoring: focal_servers"
* "skipping: no hosts matched"



**Fix:** 

1. **Define the `focal_servers` group in your Ansible inventory file**.  This file typically resides at `~/.ansible/hosts`. You'll need to add a section for `focal_servers` and list the actual IP addresses or hostnames of the servers you want to manage within that section. 


   Example:

   ```
   [focal_servers]
   server1.example.com
   server2.example.com
   ```




**Why:** Ansible uses inventory files to determine which hosts are targeted by your playbooks. The `hosts` directive in the playbook refers to a group defined within your inventory. Without a matching group, Ansible can't identify which servers should receive the commands in your playbook.
